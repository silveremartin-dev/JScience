A useful link: Matrix and Quaternion FAQ http://skal.planet-d.net/demo/matrixfaq.htm Performance: Although we have not yet finished working on the banchmarking code, we could only compare our mathematics API against a couple others. Most of the mathematics package have very incomplete matrix support except for dense double matrices. Most of the packages are also unoptimized and only target to offer a feature, not a speed optimization of a feature. Finally, some parts of our code are based on earlier libraries: namely, JAMA, Colt, JSci. Most of the time we have further optimized the code on one side, but on the other we have provided better code control (error checking, longer method names, easier methods to work with) which gives time penality. Overall, this should be a very fast library, and more than that, a library with a Java look and feel, therefore very easy to use and enhance. We will provide actual benchmarks here when available.
